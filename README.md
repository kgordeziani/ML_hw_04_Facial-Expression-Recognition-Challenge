## პირველი, ყველაზე მარტივი არქიტექტურა
ყველაზე მინიმალისტური არქიტექტურისთვის ავიღე  2 convolutional layer. ერთი 16 და მეორე 32 ფილტრით. კერნელი არის 3x3 და padding=1. თითოეულის შემდეგ მაქ აქტივაციის ლეირი ReLU და MaxPooling-საც ვაკეთებ 2x2 ზომებით. ბოლოგ მაქ FC layer, რომლელიც აბრუნებს 7 კლასის შესაბამის ლოგიტებს. 
რადგან ჩვენი ამოცანა არის მრავალკასიანი კლასიფიკაცია გადავწვიტე  CrossEntropyLoss-ის გამოყენება, ეს ფუნქცია ყველაზე ეფექტური და სტანდარტულია. Batch size-ისთის ვიფიქრე, რომ ყველაზე კარგი იქნებოდა 64-ის აღება, რადგან პატარა batch-ები პირველ რიგში ტრეინინგის დროს გაზრდიდნენ და გრადიენტებსაც უფრო "ხმაურიანს" გახდნიდნენ. ხოლო დიდი batch-ები მეტ მეხსიერებას მოითხოვენ და განზოგადებას უშლიან ხელს. 
ასევე ვაკეთებ მონაცემების არევას, shuffle=True,  ყველა ეპოქაზე, რათა მოდელმა არ დაიზეპიროს კლასების თანმიმდევრობა და თავიდან overfitting-ში არ წავიდეს.

## მეორე, შედარებით ღრმა არქიტექტურა
ზემოხსენებული პრობლემის მოსაგვარებლად გადავწყვიტე მქონოდა სამი კონვოლუციური ლეიერი და დავამატე ნორმალიზაციებიც Batch Normalization და Dropout. ამით ჩვენს მოდელს ვაძლევთ საშუელებას ისწავლოს არა მხოლოდ low-level-feature-ები, არამდენ უფრო კომპლექსური პატერენები დაიჭიროს და შესაბამისად accuracy გაზარდოს.
  * Conv(1→32, 3x3) → BatchNorm → ReLU → MaxPool   → (32, 24, 24)
  * Conv(32→64, 3x3) → BatchNorm → ReLU → MaxPool  → (64, 12, 12)
  * Conv(64→128, 3x3) → BatchNorm → ReLU → MaxPool → (128, 6, 6)
  * Flatten → Linear(4608, 512) → ReLU → Linear(512, 7)
პირველ მოდელში 16/32 ფილტრი მკაცრად ზღუდავდა ქსელს. ჩენელების რაოდენობა განსაზღვრავს, თუ რამდენი სხვადასხვა "Feature Detector" ექნება მოდელს თითოეულ დონეზე. ფილტრების გაზრდით ქსელს მივეცით საშუალება, რომ ისწავლოს გაცილებით მრავალფეროვანი და კომპლექსური ვიზუალური პატერნები. პირობითად არა მხოლოდ მარტივი ხაზები, არამედ ტუჩის კუთხის ფორმა ან ნაოჭები და ა.შ.
ასევე დავამატე ნორმალიზაციები, პირველი batch normalization - ზოგადად ღრმა ნეირონულ ქსელებში (სამი ლეიერიდან უკვე შეგვიძლია ღრმად მივიჩნიოთ) ტრეინინგის პროცესში ყოველი ფენის ინპუტების დისტრიბუცია მუდმივად იცვლება წინა ფენების პარამეტრების განახლების გამო, რაც ანელებს სწავლას და იწვევს არასტაბილურობას. batch normalization აკეთებს თითოეული მინი-ბეჩის output-ის ნორმალიზებას, რაც ასტაბილურებს ქსელს. ასევე დავამატე dropout, რათა ქსელმა კონკრეტული კომბინაციები არ დაიზეპიროს და overfitting-ში არ წავიდეს.

შედეგად მივიღთ, რომ მოდელს საერთოდ არ შესძლებია განზოგადება. ამიტომ ვხედავთ რამენად საჭიროა რეგულარიზაციის დამატება.
ამის მერე dropout ავიღ 0.5 და სახეზე გვაქ ქსელის დასტაბილურებას. Epoch 20/20 | Train Loss: 0.8225, Train Acc: 0.6739 | Val Loss: 1.2678, Val Acc: 0.5663.

გარდა ამისა ვცდი კიდევ L2 რეგულარიზაციას, ანუ გამოვიყენებ Weight Decay-ს, რომ მოდელმა დასაჯოს ზედემტად დიდი წონები, რაც ხელს უშლის ცალკეულ ნეირონებს ზედმეტი დომინირებისგან. ჰიპერპარამეტრების სხვადასხვა კომბინაციებს ვცდი და მერე საუკეთსოს ამოვარჩევ.

ვცადე სხვადასვა ვარიანტები და შედეგად დავინახე, რომ მოდელს აქვს იმის უნარი რო კარგად ისწავლოს, მაგრამ მე-15 ეპოქის შემდგომ სტატიკური Learning Rate = 0.0005 ხდება ზედმეტად უხეში, რის გამოც ოპტიმიზატორი "ახტება" ოპტიმალურ მინიმუმს და იწყება Val Loss-ის გაუარესება. ეს მიგვითითებს, რომ მოდელი გასცდა რეალური კანონზომიერებების სწავლის ფაზას და გადავიდა ხმაურის დაზეპირებაში. ფიქსირებული Learning Rate არ აძლებს ქსელს საშუალებას, უფრო მცირე და ფრთხილი ნაბიჯებით მივიდეს ლოკალურ მინიმუმში.

ამ პრობლემის მოსაგვარებლად შემოვიტანე Early Stopping. მინდა, რომ მოდელმა თავიდან იაროს შედარებით დიდი ნაბიჯებით, ხოლო როცა ვალიდაციის ლოსი "გაიჭედება" (Plateau), მან ავტომატურად შეამციროს lr, რათა მოდელს უფრო მეტი თავისუფლება ქონდეს და თვითონ იპოვოს საუკეთსო შედეგი. Early Stopping დაიცავს მოდელს Overfitting-ისგან და პროცესს დროულად გააჩერებს.

## მესამე, დაბალანსებული არქიტექტურა
წინა არქიტექტურაზე მივიღერი მოდელის გენერაციის მაქსიმუმ ~58%. ამ ამოცანი მთავარი გამოწვევა არა თუ ჰიპერპარამეტრების საუკეთესო კომბინაციის შერჩევაა, არამედ მონაცემთა კრიტიკული დისბალანსია, რის გამოც რამდენიც არ უნდა ვიწვამოთ მხოლოდ პიერპარამეტრების ოპტიმიზაციისთ ვერ მივაღწევთ საუკეთესო შედეგს.

სტანდარტულ ქსელს არ აქვს „მოტივაცია“ ისწავლოს მცირე კლასები, რადგან საერთო Loss-ში მათი წილი უმნიშვნელოა. მოდელი ხდება მიკერძებული პოპულარული კლასების მიმართ. ამ პრობლემის მოსაგვარებლად შემოვიტანეთ WeightedRandomSampler. ტრენინგის დროს უმცირესობის კლასები ხელოვნურად უფრო ხშირად (Re-sampling) ხვდება ბეჩებში. ეფექტურად, მოდელი ყოველ ეპოქაში ყველა კლასს თანაბარი სიხშირით ხედავს, რაც მკვეთრად გააუმჯობესებს თითოეული კლასის ინდივიდუალურ სიზუსტეს.

ასევე უკეთ რომ ავირიდო თავი overfitting-ისგან და დავრწმუნდე, რომ ქსელია არ დაიზეპირებს პიქსელებს დავმატე დინამიური აუგმენტაცია. მოდელი ყოველ ეპოქაში ერთი და იმავე სურათის ოდნავ განსხვავებულ, მოდიფიცირებულ ვერსიას ხედავს. ეს ხელოვნურად ზრდის დატასეტის მოცულობასა და მრავალფეროვნებას, რაც აიძულებს ქსელს ისწავლოს რეალური გეომეტრიული პატერნები (თვალების, ტუჩების ფორმა) და არა კონკრეტული პიქსელების მდებარეობა.

### შეფასება
თავიდან გამოვიყენე კლასების დაბალანსება, მაგრამ არა აუგმენტაცია.

Disgust, Fear) წონა გაზარდა და აიძულა მოდელი, მათთვის მეტი ყურადღება მიექცია. თუმცა, რადგან augmentation: none იყო, მოდელი ყოველ ეპოქაში ხედავდა ერთი და იმავე მცირე კლასის სურათების ზუსტ ასლებს. ამან გამოიწვია ის, რომ ქსელმა უბრალოდ "დაიზეპირა" უმცირესობის კლასების პიქსელები. ამის გამო Train Accuracy მე-11 ეპოქაზე 77.57%-მდე ავარდა, ხოლო Val Accuracy 55.26%-ზე გაიჭედა.
